# AGENTS.md

このリポジトリで作業する全エージェント（Claude Code / Codex / その他）への指示書。
`CLAUDE.md` はこのファイルへのシンボリックリンク。

## 永続メモリは使わない

- このリポジトリでの作業内容・ルール・方針・ユーザー嗜好などを、エージェント側の永続メモリ（例: Claude Code の `~/.claude/projects/.../memory/`）に保存しないこと。
- すべてのコンテキストはリポジトリ内（このファイル、`openspec/`、`.agents/`、`.claude/`、その他コミット対象ファイル）で管理する。
- 理由: リポジトリで一元管理することで、環境やエージェント実装をまたいでも同じ前提で作業できるようにするため。
- ユーザーから「覚えておいて」「記憶して」と言われた場合も、メモリではなくこのリポジトリ内の適切なファイル（このAGENTS.md、`openspec/`配下、専用のドキュメント等）に追記すること。

## ドキュメント・設定の能動的整備

- 作業中に再利用価値のあるルール・前提・手順が出てきたら、能動的に以下を作成・更新する:
  - `AGENTS.md`（このファイル）— エージェント横断の指示・規約
  - `.agents/` — プロジェクト固有のサブエージェント定義
  - `skills/` または `.claude/skills/` — 反復ワークフローのスキル化
  - `rules/` — コーディング規約・禁則事項などの明文化
  - `openspec/` — 仕様変更ワークフロー（OpenSpec）
- 過剰には作らない。本当に再利用価値があるものに限る。判断に迷ったらユーザーに一言確認する。

## 作業時の基本姿勢

- 不要なコメント・過剰な抽象化・将来仕様への先回りはしない。
- 破壊的・非可逆な操作（force push、ブランチ削除、本番への変更など）は事前確認する。
- リポジトリ内のファイルとgit履歴を一次情報として扱う。記憶ではなく現物を確認する。

## コミット運用

- ユーザーがアーティファクト（OpenSpec の `proposal.md` / `design.md` / `specs/**/*.md` / `tasks.md`、AGENTS.md やドキュメント更新など）の作成・変更を承認した直後に、その単位で git コミットを作成する。承認後にコミット漏れのまま次の作業へ進めない。
- OpenSpec のアーティファクト追加は 1 アーティファクト = 1 コミットを基本とする（spec ファイル群のように同時生成が自然な単位はまとめて 1 コミットでも可）。
- ローカル設定ファイル（`.claude/settings.local.json` 等）はコミット対象に含めない。
- コミットメッセージは日本語で、変更対象（例: `openspec(add-profile-site): design.md を追加`）と意図が伝わる粒度にする。
- `git add .` や `git add -A` は使わず、対象ファイルを明示してステージする。`.env` 等の機密や巨大バイナリ混入を避けるため。
