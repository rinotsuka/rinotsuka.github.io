## 書き方

企業ごとに `---` で区切る。1 企業ブロックの中に「企業エントリ」と、その下に複数の「案件エントリ」を並べる。

- 企業エントリ
  - **業界**: 客先業界（例: フィンテック、B2B SaaS）。客先名は伏せて業界名を社名表示として使う
  - **雇用形態**: 任意。例: `業務委託` / `SES` / `正社員`
  - **開始年月**: `YYYY-MM`
  - **終了年月**: `YYYY-MM` または `現在`
  - **概要**: 客先での立ち位置・総括を **2-5 行の文章** で
- 案件エントリ
  - **案件名**: 案件名
  - **ターゲット**: 任意。カンマ区切り。例: `BtoB` / `BtoC` / `BtoB, BtoC`
  - **開始年月**: `YYYY-MM`
  - **終了年月**: `YYYY-MM` または `現在`
  - **メンバー**: チーム人数（例: `5名`、`9名`）
  - **開発手法**: 任意。例: `アジャイル` / `ウォーターフォール` / `Scrum` / `Kanban`
  - **担当**: 担当した作業を **箇条書き** で
  - **技術**: カンマ区切りで列挙
  - **概要**: 案件で何をやったかを **2-5 行の文章** で

> 「概要」は MD でも JSON でも 2-5 行の文章。skill はそのままコピーするだけで要約はしない（同じ MD は何度変換しても同じ JSON になる）。
> 「担当」は箇条書きで書き、`string[]` として JSON に入れる。
> 「ターゲット」はカンマで分割して `string[]` として JSON に入れる。
